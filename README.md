# DO AI Avatar

A **Personalized AI Avatar Greeting** web application built for DigitalOcean's booth at [Cloudconf, Bangalore](https://cloudconf.ai/). Visitors fill in their details, optionally take a selfie, and receive a unique superhero avatar generated by **fal-ai/flux/schnell** running on [DigitalOcean Serverless Inference](https://docs.digitalocean.com/products/gradient-ai-platform/how-to/use-serverless-inference/).

## Features

- **Lead Capture** – Collects Name, Email ID, Company Name, and Designation into a `leads.csv` file
- **Selfie Upload** – Visitors can upload a photo (stored server-side as a reference)
- **AI Avatar Generation** – Personalised superhero avatar created via DigitalOcean Serverless Inference (`fal-ai/flux/schnell`)
- **Real-time Status Polling** – Progress bar and live status updates while the avatar is generating
- **Download** – Generated avatar can be downloaded instantly
- **Leads Export** – Admin endpoint to download the collected leads CSV

## Local Development

### Prerequisites

- Python 3.11+
- A DigitalOcean API key with Serverless Inference access

### Setup

```bash
# 1. Clone the repo
git clone https://github.com/aksprat/do-ai-avatar.git
cd do-ai-avatar

# 2. Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Set required environment variables
export DIGITALOCEAN_API_KEY="your_do_api_key_here"
export SECRET_KEY="a-random-secret-key"
export ADMIN_KEY="your-admin-key"   # Used to protect /leads/download

# 5. Run the development server
python app.py
```

Open [http://localhost:5000](http://localhost:5000).

### Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `DIGITALOCEAN_API_KEY` | ✅ | DigitalOcean API key for Serverless Inference |
| `SECRET_KEY` | ✅ | Flask session secret (use a long random string in production) |
| `ADMIN_KEY` | ⚠️ Optional | Protects the `/leads/download` endpoint |
| `PORT` | No | HTTP port (default: `5000`; App Platform sets this automatically) |
| `DEBUG` | No | Set to `true` for development hot-reload (never in production) |

## Deployment on DigitalOcean App Platform

1. Fork / push this repo to GitHub.
2. Go to the [DigitalOcean Control Panel](https://cloud.digitalocean.com/apps) → **Create App**.
3. Select the GitHub repository and the `main` branch.
4. Choose **Python** as the environment.
5. Set **Build Command**: `pip install -r requirements.txt`
6. Set **Run Command**: `gunicorn app:app --workers 2 --threads 4 --bind 0.0.0.0:$PORT`
7. Add the required environment variables (`DIGITALOCEAN_API_KEY`, `SECRET_KEY`, `ADMIN_KEY`) as **encrypted secrets**.
8. Deploy 🚀

> Alternatively, the `.do/app.yaml` spec file is included for one-click App Platform deployment.

## API Reference

### `GET /`
Lead capture and selfie upload form.

### `POST /generate`
Submits the lead form. Saves the lead to `leads.csv`, starts async avatar generation, and redirects to the result page.

**Form fields**: `name`, `email`, `company`, `designation`, `photo` (file, optional)

### `GET /result/<request_id>?name=<name>`
Result page. Polls `/api/status/<request_id>` via JavaScript until the avatar is ready.

### `GET /api/status/<request_id>`
Proxies the DigitalOcean Serverless Inference status endpoint. Returns JSON with `status` and `output.images`.

### `GET /leads/download?key=<ADMIN_KEY>`
Downloads the collected `leads.csv` file. Pass `?key=<ADMIN_KEY>` if `ADMIN_KEY` is set.
